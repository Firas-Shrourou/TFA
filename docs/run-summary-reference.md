# Run Summary Reference

`run_results_summary.json` is the compact audit record for a completed TFA run.
It records the frozen input contract, the pipeline call log, and the scientific
summary blocks written by each specialist.

The file is created in every timestamped run folder:

```text
results/<run-folder>/run_results_summary.json
```

For package-root runs in the development layout, the same file appears under:

```text
tfa-results/runs/<run-folder>/run_results_summary.json
```

## Purpose

Use `run_results_summary.json` when you need to answer:

- What scalar route was requested?
- Which settings file was frozen into the run folder?
- Which specialists ran, in what order, and with what status?
- What acoustic-normalized Hubble history did the route produce?
- Did the physical guards pass?
- Which plots, BAO products, and RSD products were written?
- Which fields should be combined across many runs for comparison?

The summary is not a replacement for the CSV products. It is the high-level
index and audit record. Per-redshift and per-datum numerical detail remains in
the CSV files named by the summary.

## Top-level shape

Current summaries use:

```json
{
  "summary_schema_version": "0.1",
  "run": {},
  "contract": {},
  "calls": [],
  "results": {}
}
```

| Field | Generated by | Meaning | Used by |
|---|---|---|---|
| `summary_schema_version` | `tfa_common` | Version of the summary JSON layout. | Readers and aggregation tools. |
| `run` | `tfa_common` | Run identity, folder metadata, hub identity, and final run status. | All readers; useful for reproducibility. |
| `contract` | `tfa_common` | Compact copy of the user route definition from the frozen settings. | Specialists use it for labels and sanity checks; users use it for audit. |
| `calls` | `tfa_common` | Ordered call log for each specialist. | Debugging, runtime audit, and failure review. |
| `results` | Specialist modules | Scientific and file-output summary blocks. | Plot, BAO, RSD, combiner, and human readers. |

## `run`

Generated by `tfa_common` when the run folder is created and finalized.

| Field | Meaning |
|---|---|
| `run_id` | UUID-like unique identifier for the run. |
| `created_utc` | UTC timestamp when the run summary was initialized. |
| `prefix` | Run-folder prefix from settings, usually `tfa`. |
| `run_folder` | Folder name containing this run's files. |
| `tfa_project_release` | Package release recorded by the hub. |
| `hub` | Script identity for the `tfa_common` build that orchestrated the run. |
| `frozen_settings_file` | Filename of the copied settings contract, normally `environment-settings.json`. |
| `status` | `initialized`, `completed`, or `completed_with_errors`. |

`run.status` is finalized by `tfa_common` after all specialists have returned.
Only the acoustic and physics-guard specialists are fatal for the overall run
code. Plot, BAO, and RSD failures are recorded but do not by themselves make
the hub return `Error`.

## Script identity blocks

Several sections contain a `script` object:

```json
"script": {
  "tfa_project_release": "0.0.4",
  "script_name": "tfa_acoustic_validator",
  "script_version": "0.1.5",
  "script_build": "0001",
  "script_api_version": "0.1",
  "settings_schema_version": "0.1"
}
```

The exact fields vary slightly by specialist. The identity block records which
script wrote that result section. Use `RELEASE-NOTES.md` for historical build
details.

## `contract`

Generated by `tfa_common` from the parsed settings before the specialists run.
It is a compact copy of `user_adjustable.potential`, plus the frozen settings
source.

| Field | Meaning |
|---|---|
| `benchmark_id` | Human-readable route label used in outputs and plots. |
| `V_of_phi` | Potential expression string used by the run. |
| `dV_dphi` | Analytic derivative expression string, if supplied. |
| `parameters` | Numeric parameter object referenced by the expressions. |
| `initial_phi` | Initial scalar-field value. |
| `initial_phi_N` | Initial field derivative `dphi/dN`. |
| `user_remarks` | Optional user note copied from settings. |
| `settings_source` | Frozen settings filename inside the run folder. |

The specialists rebuild runtime objects from the frozen
`environment-settings.json`, not from this compact block alone. The contract
block is primarily an audit and labeling record.

## `calls`

Generated by `tfa_common`. Each entry is appended immediately after a specialist
returns.

| Field | Meaning |
|---|---|
| `specialist` | Module called by the hub. |
| `code` | `"OK"` or `"Error"` as returned by that specialist. |
| `desc` | Human-readable status message. |
| `started_utc` | UTC timestamp before the call. |
| `ended_utc` | UTC timestamp after the call. |
| `duration_s` | Wall-clock duration in seconds, rounded by the hub. |

Current call order:

```text
tfa_acoustic_validator
tfa_physics_guard_validator
tfa_plot_exporter
tfa_bao_validator
tfa_rsd_validator
```

## `results.acoustic_validator`

Generated by `tfa_acoustic_validator`. This is the central scientific block.
Plot, BAO, and RSD modules consume values from this block.

| Field | Meaning | Used by |
|---|---|---|
| `status` | Acoustic validator status, usually `OK`. | Readers. |
| `script` | Script identity for the acoustic validator. | Readers, provenance. |
| `H0_X_kms` | Route-specific acoustic-preserving Hubble constant in km/s/Mpc. | Plot labels, BAO, RSD, comparison tables. |
| `delta_X` | Relative offset of `H0_X_kms` from `H0_ref_kms`: `(H0_X - H0_ref) / H0_ref`. | CSV metadata, plots, route comparison. |
| `band` | Acoustic-band classification: `STRICT`, `LOOSE_2S`, `LOOSE_3S`, or `EXCLUDED`. | Export gate, plots, route comparison. |
| `H0_ref_kms` | Reference Hubble constant from settings. | Metadata and comparison. |
| `Omega_m_X` | Matter fraction rescaled to the route's acoustic-normalized Hubble constant. | RSD growth calculation context. |
| `D_M_X_Mpc` | Route transverse comoving distance to the acoustic scale. | Acoustic audit. |
| `D_M_LCDM_Mpc` | Reference Lambda-CDM transverse comoving distance to the acoustic scale. | Acoustic audit. |
| `trajectory_file` | Filename of the full scalar trajectory CSV. | Plot exporter, physics guards. |
| `trajectory_rows` | Number of rows written to `trajectory.csv`. | Audit. |
| `export_gate` | Gate state deciding whether normalized histories are exported. | Plot, BAO, RSD. |
| `shape_check` | `PASS`/`FAIL`/`null` check for `E_X(0) = 1`. | Readers; downstream diagnostics expect PASS when histories exist. |
| `normalization_check` | `PASS`/`FAIL`/`null` check for `H_X(0) = H0_X`. | BAO and RSD also perform their own checks. |
| `expansion_history_files` | Filenames for exported history CSV products, or `null` when gated. | Plot, BAO, RSD. |
| `energy_fractions` | Low-redshift energy-fraction summary. | Plot exporter energy-fraction plot, readers. |

### `acoustic_anchor`

Generated by the acoustic validator.

| Field | Meaning | Used by |
|---|---|---|
| `z_star` | Photon-decoupling redshift used for the acoustic angular scale. | Acoustic audit and plots. |
| `r_s_raw_Mpc` | Raw sound horizon before calibration. | Acoustic audit. |
| `rs_calibration` | Calibration factor applied to the raw sound horizon. | Acoustic audit. |
| `r_s_Mpc` | Calibrated sound horizon at `z_star`. | RSD uses this as a sentinel that acoustic output exists. |
| `D_M_LCDM_Mpc` | Reference Lambda-CDM distance to `z_star`. | Acoustic audit. |
| `theta_star` | Target acoustic angular scale. | Acoustic audit. |
| `anchor_check` | Check string for the acoustic anchor, usually `PASS`. | Readers. |
| `z_drag` | Drag-epoch redshift. | BAO ruler audit. |
| `r_drag_Mpc` | Drag-epoch sound horizon. | BAO validator as the BAO ruler. |

BAO uses `r_drag_Mpc`, not `r_s_Mpc`, because DESI BAO ratios are expressed
with the drag-epoch ruler.

### `export_gate`

Generated by the acoustic validator from settings and the derived `band`.

| Field | Meaning |
|---|---|
| `enabled` | Whether the gate is active. |
| `accepted` | Whether this route's band is accepted for normalized-history export. |
| `band` | The derived acoustic band. |
| `accepted_bands` | Bands allowed by the settings contract. |

When `accepted` is `false`, `expansion_history_files` entries are `null`, BAO
and RSD are skipped, and `H_of_z`/`delta_H` plots are skipped.

### `expansion_history_files`

Generated by the acoustic validator.

| Field | Meaning |
|---|---|
| `shape` | `expansion_history_shape.csv`, containing `E_X(z)`, or `null` if gated. |
| `normalized_history` | `expansion_history_h0x_normalized.csv`, containing `H_X(z)`, or `null` if gated. |
| `w_of_z` | `w_of_z.csv`, or absent/null in older or gated runs. |

### `energy_fractions`

Generated by acoustic validator v0.1.5 or newer.

| Field | Meaning |
|---|---|
| `Omega_phi_0` | Scalar-field density fraction at `z=0` from the ODE trajectory. |
| `Omega_m_0` | Route matter fraction at `z=0`, computed from the route budget. |
| `z_eq_lcdm` | Reference Lambda-CDM matter/dark-energy equality redshift. |
| `z_eq_route` | Route redshift where `Omega_phi(z) = 0.5`, if found. |
| `delta_z_eq` | `z_eq_route - z_eq_lcdm`, if a crossing is found. |
| `z_eq_note` | Optional note when no route equality crossing is found. |

The plot exporter reads this block to write `energy_fractions.png/pdf`.

## `results.physics_guard_validator`

Generated by `tfa_physics_guard_validator`. It reads `trajectory.csv` and the
frozen settings. It does not require the normalized expansion-history CSVs.

| Field | Meaning | Used by |
|---|---|---|
| `status` | Guard validator status, usually `OK`. | Readers. |
| `script` | Script identity for the guard validator. | Provenance. |
| `overall_pass` | True only if all four guard verdicts pass. | Run assessment. |
| `verdicts` | Compact boolean verdicts for each guard. | Readers and combiner. |
| `diagnostics` | Numeric values behind the guard decisions. | Readers and combiner. |
| `thresholds` | Guard thresholds read from frozen settings. | Reproducibility audit. |
| `guards_csv` | Filename of the row-level guard CSV. | Readers. |
| `guards_csv_rows` | Number of rows written to the guard CSV. | Audit. |

### `verdicts`

| Field | Meaning |
|---|---|
| `canonical_ok` | Whether `w_phi` respects the canonical floor. |
| `canonical_thawing_ok` | Whether the late-time route satisfies the thawing monotonicity check. |
| `phantom_crossing_ok` | Whether the route avoids phantom crossing below the configured tolerance. |
| `BBN_OK` | Whether the scalar density fraction at BBN is below the configured bound. |

### `diagnostics`

| Field | Meaning |
|---|---|
| `minimum_w_phi` | Minimum scalar equation-of-state value in the checked history. |
| `minimum_delta_w` | Minimum late-time monotonicity diagnostic used by the thawing check. |
| `late_sample_count` | Number of late-time trajectory samples used by the thawing check. |
| `Omega_phi_BBN` | Frozen-field scalar density fraction at `bbn_z`. |

## `results.plot_exporter`

Generated by `tfa_plot_exporter`. It reads `trajectory.csv`, the frozen
settings, and the acoustic result block. It reads
`expansion_history_h0x_normalized.csv` when that file exists.

| Field | Meaning |
|---|---|
| `status` | Plot exporter status, usually `OK`. |
| `script` | Script identity for the plot exporter. |
| `plots_written` | List of PNG/PDF files written by the exporter. |
| `plots_skipped` | Plot stems skipped because an optional input was absent. |
| `plot_count` | Count of plot stems written, not count of individual PNG/PDF files. |

Always-available plots use `trajectory.csv`: `w_of_z`, `Omega_phi`, and
`phase_portrait`. The `energy_fractions` plot additionally requires the
`results.acoustic_validator.energy_fractions` block. `H_of_z` and `delta_H`
require `expansion_history_h0x_normalized.csv` and are skipped when the export
gate rejects the route.

## `results.bao_validator`

Generated by `tfa_bao_validator`. It is non-fatal from the hub's perspective.

The BAO validator reads:

- `environment-settings.json`
- `run_results_summary.json`
- `expansion_history_h0x_normalized.csv`
- `results.acoustic_validator.acoustic_anchor.r_drag_Mpc`

If disabled or gated, it writes a minimal skipped block:

| Field | Meaning |
|---|---|
| `status` | `skipped`. |
| `reason` | Skip reason, such as `route not accepted by export gate`. |
| `script` | Script identity. |

When successful, it writes:

| Field | Meaning |
|---|---|
| `status` | `OK`. |
| `script` | Script identity for the BAO validator. |
| `dataset_label` | BAO dataset identifier. |
| `rd_X_Mpc` | Route drag-epoch ruler used for BAO ratios. |
| `rd_source` | Source field for `rd_X_Mpc`, currently `acoustic_anchor.r_drag_Mpc`. |
| `geometry` | Geometry assumption, currently `flat`. |
| `Omega_K` | Curvature diagnostic; must be near zero. |
| `datum_count` | Number of BAO data points. |
| `chi2` | Full-covariance chi-squared. |
| `dof` | Degrees of freedom used for reduced chi-squared. |
| `reduced_chi2` | `chi2 / dof`. |
| `chi2_diag` | Diagonal-only chi-squared from per-datum pulls. |
| `reduced_chi2_diag` | `chi2_diag / dof`. |
| `max_abs_pull` | Largest absolute per-datum pull. |
| `n_within_1sigma` | Count of data points within one sigma. |
| `n_within_2sigma` | Count within two sigma. |
| `n_within_3sigma` | Count within three sigma. |
| `n_outside_3sigma` | Count outside three sigma. |
| `bao_results_per_datum_csv` | Row-level BAO output CSV. |
| `bao_pulls_plot` | BAO pull plot PNG filename. |
| `interpolation_method` | Interpolation method used for `H_X(z)`, currently `PchipInterpolator`. |
| `normalization_check` | Check that the normalized-history CSV satisfies its contract. |

## `results.rsd_validator`

Generated by `tfa_rsd_validator`. It is non-fatal from the hub's perspective.

The RSD validator reads:

- `environment-settings.json`
- `run_results_summary.json`
- `expansion_history_h0x_normalized.csv`
- `results.acoustic_validator.H0_X_kms`
- `results.acoustic_validator.acoustic_anchor.r_s_Mpc` as an acoustic-run sentinel

If disabled or gated, it writes a minimal skipped block:

| Field | Meaning |
|---|---|
| `status` | `skipped`. |
| `reason` | Skip reason, such as `route not accepted by export gate`. |
| `script` | Script identity. |

When successful, it writes:

| Field | Meaning |
|---|---|
| `status` | `OK`. |
| `script` | Script identity for the RSD validator. |
| `dataset_label` | RSD dataset identifier. |
| `datum_count` | Number of RSD data points. |
| `sigma8_reference` | External reference sigma8 normalization. |
| `sigma8_X` | Route sigma8 after applying the growth-ratio normalization. |
| `growth_ratio_D_X_over_D_LCDM` | Ratio of route growth amplitude to reference growth amplitude. |
| `chi2` | Chi-squared against the RSD compilation. |
| `dof` | Degrees of freedom used for reduced chi-squared. |
| `reduced_chi2` | `chi2 / dof`. |
| `max_abs_pull` | Largest absolute per-datum pull. |
| `n_within_1sigma` | Count of data points within one sigma. |
| `n_within_2sigma` | Count within two sigma. |
| `n_within_3sigma` | Count within three sigma. |
| `n_outside_3sigma` | Count outside three sigma. |
| `rsd_results_per_datum_csv` | Row-level RSD output CSV. |
| `rsd_pulls_plot` | RSD pull plot PNG filename. |
| `rsd_growth_plot` | RSD growth-curve plot PNG filename. |
| `growth_z_start` | Starting redshift for the growth integration. |
| `ode_solver` | ODE method used for growth integration. |
| `normalization_check` | Check that the normalized-history CSV satisfies its contract. |

## Producer and consumer map

| Producer | Writes | Read by |
|---|---|---|
| `tfa_common` | `summary_schema_version`, `run`, `contract`, `calls`, empty `results`; finalizes `run.status`. | All specialists and readers. |
| `tfa_acoustic_validator` | `results.acoustic_validator`, `trajectory.csv`, optional normalized-history CSVs. | Physics guard, plot exporter, BAO, RSD, readers. |
| `tfa_physics_guard_validator` | `results.physics_guard_validator`, `physics_guards.csv`. | Readers and summary combiner. |
| `tfa_plot_exporter` | `results.plot_exporter`, plot PNG/PDF files. | Readers. |
| `tfa_bao_validator` | `results.bao_validator`, `bao_results_per_datum.csv`, BAO pull plot. | Readers and summary combiner. |
| `tfa_rsd_validator` | `results.rsd_validator`, `rsd_results_per_datum.csv`, RSD plots. | Readers and summary combiner. |
| `tfa_combined_csv_results` | Does not modify run summaries; reads many summaries into one union CSV. | External analysis tools. |

## Gated and optional fields

Not every run has every successful result block.

Common cases:

- If the export gate rejects a route, normalized-history CSVs are not written.
- When normalized histories are absent, BAO and RSD write `status: "skipped"`.
- When normalized histories are absent, `H_of_z` and `delta_H` plots are skipped.
- If `energy_fractions` is absent in an older run, the energy-fractions plot is skipped.
- If a validator is disabled in settings, its result block records a skipped status and reason.

Consumers should check `status` before assuming success-only fields exist.

## Recommended reading workflow

For a single run:

1. Read `contract.benchmark_id`, `V_of_phi`, `parameters`, and initial field data.
2. Check `run.status` and the `calls` array.
3. Read `results.acoustic_validator.H0_X_kms`, `band`, and `export_gate.accepted`.
4. Check `results.physics_guard_validator.overall_pass`.
5. If present and `status` is `OK`, inspect BAO and RSD chi-squared summaries.
6. Follow named CSV files for row-level numerical detail.

For many runs:

1. Use `docs/summary-combiner.md`.
2. Combine summaries into one CSV.
3. Treat runtime metadata fields such as `run_id`, timestamps, and folders as
   expected differences between repeated runs.
4. Compare scientific fields such as `H0_X_kms`, guard verdicts, BAO summaries,
   RSD summaries, and energy-fraction values.
