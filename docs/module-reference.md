# TFA Module Reference

This document summarizes the public behavior of the active TFA pipeline modules.
Release history and superseded builds are documented separately in
`RELEASE-NOTES.md`.

## Pipeline overview

The package-root launcher calls the approved `tfa_common` build. That hub reads
the package-root `tfa-environment-settings.json`, creates a timestamped run
folder, freezes the input settings into that folder, initializes
`run_results_summary.json`, and runs the active specialist modules in sequence.

The current pipeline is:

```text
tfa_common
-> tfa_acoustic_validator
-> tfa_physics_guard_validator
-> tfa_plot_exporter
-> tfa_bao_validator
-> tfa_rsd_validator
```

Plot, BAO, and RSD products depend on the files written earlier in the run. The
run folder is the unit of reproducibility: it contains the frozen input
contract, all numerical products, the summary record, and the plots generated by
the specialists.

## tfa_common

`tfa_common` is the pipeline hub. It resolves approved specialist builds,
creates the run folder, freezes settings, calls each specialist, and records
status information in `run_results_summary.json`.

Current approved build:

```text
scripts/tfa_common/v0.9.1-build.0002/
```

Primary entry point:

```python
import tfa_common as tfa

code, desc = tfa.run()
```

The package-root launchers call this entry point automatically.

## tfa_acoustic_validator

`tfa_acoustic_validator` is the scalar-field engine and acoustic normalizer. It
evaluates the potential from `user_adjustable.potential`, integrates the
homogeneous scalar-field route, computes the dimensionless expansion shape
`E_X(z)`, solves for the acoustic-preserving `H0_X`, and writes the normalized
history `H_X(z)`.

Current approved build:

```text
scripts/tfa_acoustic_validator/v0.1.5-build.0001/
```

Public entry point:

```python
from tfa_acoustic_validator import run_acoustic_validator

code, desc = run_acoustic_validator(run_folder)
```

Main outputs:

| File | Contents |
|---|---|
| `trajectory.csv` | Full scalar trajectory and state history. |
| `expansion_history_shape.csv` | Dimensionless route shape `E_X(z)`. |
| `expansion_history_h0x_normalized.csv` | Normalized Hubble history `H_X(z)`. |
| `w_of_z.csv` | Scalar equation-of-state history. |
| `run_results_summary.json` | Acoustic anchor, route metadata, verdicts, and energy-fraction summary. |

The acoustic validator also computes the drag-epoch sound horizon `r_drag_Mpc`
used by the BAO validator.

## tfa_physics_guard_validator

`tfa_physics_guard_validator` checks whether the integrated route satisfies the
package's physical-admissibility guards. It reads `trajectory.csv` and the
frozen settings from the run folder, applies the configured thresholds, writes a
long-form guard table, and enriches the summary JSON.

Current approved build:

```text
scripts/tfa_physics_guard_validator/v0.1.4-build.0001/
```

Public entry point:

```python
from tfa_physics_guard_validator import run_physics_guard_validator

code, desc = run_physics_guard_validator(run_folder)
```

Guards:

| Guard | Purpose |
|---|---|
| `canonical` | Checks that `w_phi` remains above the canonical floor. |
| `thawing` | Checks late-time thawing monotonicity. |
| `phantom_crossing` | Checks for crossing below the phantom boundary. |
| `BBN` | Checks the scalar density fraction against the BBN bound. |

Main output:

```text
physics_guards.csv
```

## tfa_plot_exporter

`tfa_plot_exporter` generates diagnostic plots from run-folder products. It is a
consumer module: it does not recompute the scalar dynamics.

Current approved build:

```text
scripts/tfa_plot_exporter/v0.1.1-build.0001/
```

Public entry point:

```python
from tfa_plot_exporter import run_plot_exporter

code, desc = run_plot_exporter(run_folder)
```

Plot products:

| File | Source data |
|---|---|
| `w_of_z.png/pdf` | `trajectory.csv` |
| `Omega_phi.png/pdf` | `trajectory.csv` |
| `phase_portrait.png/pdf` | `trajectory.csv` |
| `H_of_z.png/pdf` | `expansion_history_h0x_normalized.csv` |
| `delta_H.png/pdf` | `expansion_history_h0x_normalized.csv` |
| `energy_fractions.png/pdf` | `trajectory.csv` plus the energy-fraction summary block |

Plots that require the normalized expansion history are skipped when the export
gate rejects a route and the required CSV is absent.

## tfa_bao_validator

`tfa_bao_validator` evaluates BAO distance diagnostics for a completed run. It
reads the normalized expansion history and acoustic anchor, computes the
relevant distance ratios with the route's own drag-epoch sound horizon, and
compares them against the bundled DESI DR2 ALL GCcomb BAO data.

Current approved build:

```text
scripts/tfa_bao_validator/v0.1.1-build.0001/
```

Public entry point:

```python
from tfa_bao_validator import run_bao_validator

code, desc = run_bao_validator(run_folder)
```

Computed ratios:

```text
D_H(z) / r_drag
D_M(z) / r_drag
D_V(z) / r_drag
```

Outputs:

| File | Contents |
|---|---|
| `bao_results_per_datum.csv` | Row-level model values, residuals, pulls, and distance quantities. |
| `bao_pulls.png/pdf` | Pull chart for the BAO data vector. |
| `run_results_summary.json` | BAO summary statistics. |

## tfa_rsd_validator

`tfa_rsd_validator` evaluates growth-rate diagnostics for a completed run. It
reads `expansion_history_h0x_normalized.csv`, evolves the linear growth factor
inside the route's normalized history, computes `f_sigma8(z)`, and compares the
route against the bundled 18-point redshift-space distortion compilation.

Current approved build:

```text
scripts/tfa_rsd_validator/v0.1.0-build.0001/
```

Public entry point:

```python
from tfa_rsd_validator import run_rsd_validator

code, desc = run_rsd_validator(run_folder)
```

Outputs:

| File | Contents |
|---|---|
| `rsd_results_per_datum.csv` | Row-level model values, residuals, pulls, and growth quantities. |
| `rsd_pulls.png/pdf` | Pull chart for the RSD measurements. |
| `rsd_growth.png/pdf` | Continuous `f_sigma8(z)` curve with data points and reference curve. |
| `run_results_summary.json` | RSD summary statistics. |

The bundled RSD data live in `data/fsigma8_gold/`.
